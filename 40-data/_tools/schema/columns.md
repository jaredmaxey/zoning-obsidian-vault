# CSV column specifications

Authoritative column definitions for the `40-data/` CSVs. See `_CONVENTIONS.md` §9 for the canonical/derived rules and the provenance/confidence vocabulary.

Shared enums:
- **provenance**: `official-pdf` | `official-html` | `secondary-mirror` | `inferred` | `unknown`
- **confidence**: `verified-human` | `extracted` | `low` | `tbd`
- **category**: `residential` | `commercial` | `industrial` | `mixed-use` | `agricultural` | `special` | `downtown` | `planned` | `open-space` | `overlay`
- **Numeric fields** hold a number, an empty cell (unknown/not extracted), or the literal `TBD`. Never guess a number.
- Local PDF references resolve against `../Maricopa County development standards/` (the primary-source library). `code_url` may be a URL or a local PDF filename.

---

## jurisdictions.csv — CANONICAL
One row per jurisdiction in the Maricopa primary-source set.

| column | meaning |
|--------|---------|
| jurisdiction_id | `az-{slug}` primary key |
| name | Human name (e.g., "City of Phoenix") |
| level | `state` \| `county` \| `city` (towns recorded as `city`; noted in `notes`) |
| county | County name |
| state | Two-letter postal |
| incorporated | `yes` for municipalities; `no` for the county (unincorporated) row |
| population | Integer or `TBD` |
| zoning_code_name | Title of the governing zoning code |
| code_host | Municode \| General Code \| American Legal \| Code Publishing \| Municipal Code Online \| City website \| PDF |
| code_url | URL to the live code, or local PDF filename |
| general_plan | General-plan PDF filename or URL |
| engineering_standards_doc | Engineering/design-standards doc filename or URL (blank if none) |
| vault_path | Vault-relative path to the jurisdiction's `00-overview.md` (or county note) |
| notes | Free text |

## source_registry.csv — CANONICAL
One row per primary-source document (machine-readable form of `Maricopa County development standards/INDEX.html`).

| column | meaning |
|--------|---------|
| source_id | `{jurisdiction_id}-{doc_type}` primary key (referenced by zone records' `source_doc`) |
| jurisdiction_id | FK to jurisdictions.csv |
| doc_type | `zoning` \| `subdivision` \| `comprehensive-plan` \| `general-plan` \| `engineering` \| `dev-standards` \| `other` |
| title | Document title |
| format | `pdf` \| `link` |
| local_file | PDF filename (blank for links) |
| url | URL (blank for local PDFs) |
| host | Publishing platform / host |
| compiled_date | Date the index was compiled (from INDEX.html) |
| last_verified | Date a human last confirmed the source is live (blank until verified) |
| notes | Free text |

## parking_standards.csv — CANONICAL
Parking minimums by use × jurisdiction. Feeds `30-reference/parking-ratios-by-use.md` (regenerated by sync.py).

| column | meaning |
|--------|---------|
| jurisdiction_id | FK to jurisdictions.csv |
| use_type | e.g., `multifamily`, `office`, `retail`, `industrial`, `restaurant` |
| ratio | e.g., `1.5`, `2.0` (numeric portion) |
| unit_basis | e.g., `per unit`, `per 1,000 sf GFA` |
| source_citation | Ordinance section/table |
| provenance | see shared enums |
| confidence | see shared enums |
| human_verified | YYYY-MM-DD or blank (AI leaves blank) |

## zoning_standards.csv — GENERATED (do not hand-edit)
Flat projection of every zone in `zones/*.json`, produced by `sync.py`. One row per zone.

Columns: `zone_id, jurisdiction_id, jurisdiction_name, zone_code, zone_name, category, min_lot_size_sf, min_lot_width_ft, max_density_dua, max_far, max_lot_coverage_pct, max_height_ft, max_stories, front_setback_ft, side_setback_ft, rear_setback_ft, parking_ratio, source_doc, source_citation, source_page, provenance, confidence, human_verified, notes`
